Parallel Programming: CUDA Gradient Calculation

Designed and implemented a high-performance GPU-based solution for computing gradients in multivariable equation systems using CUDA and C.

Key Technical Contributions

  • Massive Parallelism: Implemented 1D/2D grid mapping where each GPU thread calculates an independent cell of the Jacobian matrix, achieving "embarrassingly parallel" workload distribution.
  • Hardware Optimization: Configured optimal block sizes (1024 threads) to maximize GPU occupancy and efficient warp scheduling.
  • Scalability: Architecture supports systems with thousands of functions and variables, demonstrating GPGPU effectiveness for scientific computing.

Technologies Used

  • CUDA (Compute Unified Device Architecture)
  • C Programming Language
  • GPU Computing

Distributed Applications: University Management System

Developed a distributed Client-Server application in Java for managing university data (Degrees, Subjects, and Enrollments) using TCP Sockets with multi-threaded architecture.

Architectural Highlights

  • Client-Server Model: Multi-threaded server handling concurrent clients with custom communication protocol.
  • Data Persistence: Singleton Pattern implementation for consistent data access across all client threads.
  • Multithreading: Each new connection spawns a ClientHandler thread, enabling non-blocking multi-user service.

Core Features

  • User authentication and session management
  • CRUD operations (Create, Read, Update, Delete) for Degrees and Enrollments
  • Robust 3-step handshake protocol for secure object transfer over separate data channels
  • HTTP-style error handling (404, 401, 200 status codes)

Technologies Used

  • Java Sockets (TCP/IP)
  • Object Serialization
  • Multi-threading
  • Design Patterns (Singleton, Command Pattern)
